1. 项目概述
本系统基于 树莓派（Raspberry Pi） 开发板，融合了深度学习目标检测、物联网通信、传感器数据采集与Web 可视化控制等技术，实现了一套面向农业害虫监测与智能控制的完整解决方案。系统能够实时识别田间害虫，采集环境参数（温湿度、土壤湿度、超声波距离、雨滴状态），通过 MQTT 协议上报至云平台（OneNET），并支持远程控制继电器（水泵/喷雾器）及双舵机（云台、喷洒装置）。同时，系统提供了基于 Flask 的 Web 界面，用户可通过浏览器实时查看带检测框的视频流、调节检测阈值、控制舵机角度以及手动/自动控制继电器。

2. 系统整体架构
系统采用 边缘计算 + 云平台 的混合架构，主要分为三层：

感知层：树莓派搭载 CSI 摄像头（OV5647）、DHT11 温湿度传感器、PCF8591 土壤湿度传感器、HC-SR04 超声波测距模块、数字雨滴传感器，以及执行设备（继电器、PCA9685 驱动的双舵机）。

边缘计算层：树莓派运行 YOLOv5 ONNX 模型进行实时害虫检测；运行多线程程序同时处理视频流、传感器读取、MQTT 通信和 Web 服务。

应用层：

本地 Web 界面（Flask + MJPEG 流）提供视频监控、传感器数据显示、控制面板。

OneNET 云平台接收 MQTT 上报数据，并支持下发继电器控制指令。

整体数据流：

摄像头采集图像 → YOLOv5 推理 → 绘制检测框 → MJPEG 流推送至前端。

传感器定时读取 → 更新全局变量 → 前端轮询显示 → MQTT 上报云端。

用户通过 Web 界面或云端下发指令 → 树莓派执行（舵机角度/继电器开关）。

3. 软件模块设计
4.1 YOLOv5 害虫识别模块
模型训练：在 PC 上使用自定义害虫数据集训练 YOLOv5s 模型，导出为 ONNX 格式（best.onnx）。

推理线程：树莓派上使用 onnxruntime 加载模型，输入尺寸 640×640，每 2 帧推理一次（跳帧优化），输出检测框坐标、类别、置信度。

运动检测：可选帧差法运动检测，仅在画面有变化时触发推理，降低 CPU 负载。

后处理：包括非极大值抑制（NMS）、坐标缩放映射、置信度过滤。

绘制与推流：在原始图像上绘制红框和标签，再通过 Flask 的 /video_feed 路由以 MJPEG 格式输出。

3.2 传感器采集模块
多线程读取：独立线程每 2 秒读取所有传感器（DHT11、土壤、超声波、雨滴），避免阻塞视频流。

异常处理：DHT11 采用重试机制，读取失败时跳过本次上报，保证程序稳定。

数据共享：使用全局变量 latest_sensor 存储最新值，供 Web API 和 MQTT 上报使用。

3.3 MQTT 通信模块
平台：中国移动 OneNET Studio。

功能：

上报属性：温度、湿度、土壤湿度、距离、雨滴状态。

订阅指令：接收云端下发的 relay 控制消息，解析后控制继电器开关。

实现：使用 paho-mqtt 库，独立线程运行 loop_forever()，保证长连接。

3.4 前端界面
技术：HTML + CSS + 原生 JavaScript，无第三方框架。

主要功能：

实时显示 MJPEG 视频流。

显示传感器数据（温度、湿度、土壤湿度、距离、雨滴状态）。

滑块调节置信度阈值，实时生效。

两个滑块分别控制双舵机角度（0~180°）。

手动继电器开关按钮。

自动模式（可选）：设置土壤湿度阈值，低于阈值自动开启水泵（后续扩展）。

通信方式：使用 fetch API 轮询 /api/sensor 和 /api/status，每 2 秒更新；舵机和继电器控制使用 POST 请求即时发送。
